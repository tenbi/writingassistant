# Display Name Strategy

## 背景

現在のアプリは、X / Threads / Instagram / TikTok の投稿 URL から以下を生成する構成になっている。

- `platform`
- `normalizedUrl`
- `userId`
- `userName`
- `profileUrl`

X は公開 `oEmbed` を使うことで、投稿 URL から `author_name` を比較的安定して取得できる。

一方で Threads / Instagram は、匿名のサーバーサイド取得だけで表示名を安定抽出するのが難しい。
公開 HTML には表示名相当が含まれるケースもあるが、取得条件や返却内容が不安定で、現状の route 実行経路では handle fallback になることがある。

## 方式1: ログイン済みブラウザ経由で取得する

### 概要

ユーザーのローカルブラウザで、ログイン済みセッションを使って Threads / Instagram の投稿ページまたはプロフィールページにアクセスし、表示名をクライアント側で取得する方式。

### 具体的な実装イメージ

1. `app/page.tsx` で「ブラウザ取得モード」を用意する
2. クライアント側 fetch もしくはブラウザ自動化で対象 URL にアクセスする
3. DOM / meta / 埋め込み JSON から表示名を抽出する
4. 結果を UI に反映し、その後 `/api/render-template` に渡す

### 実装候補

- ブラウザ拡張を作る
  - content script で Threads / Instagram ページから `user_name` を読む
  - アプリ本体には `postMessage` などで渡す
- Playwright / Puppeteer をローカル起動する
  - ログイン済みブラウザプロファイルを使う
  - ページ遷移後に DOM や meta を読む
- クライアントで直接 `window.open` した別画面から取得する
  - ただし cross-origin 制限でかなり厳しい

### メリット

- Threads / Instagram の表示名取得精度を最も上げやすい
- 実際にユーザーが見ているページ情報を使える
- 投稿 URL ベースの要件と相性が良い

### デメリット

- 実装コストが高い
- セッションやブラウザ依存が増える
- 配布形態によってはブラウザ拡張や自動化権限が必要

### この方式が向いているケース

- 「表示名自動取得」は絶対に落としたくない
- ローカル専用ツールでもよい
- ブラウザ依存を受け入れられる

## 方式2: サーバー側ベストエフォート + 手動補完前提

### 概要

現在の延長線上の方式。匿名で取得できる公開 HTML / meta / 埋め込み情報から表示名を取れるだけ取って、取れないときは UI 上で手動修正させる。

### 具体的な実装イメージ

1. `/api/resolve-social` で URL 解析
2. platform ごとに公開情報から `userId`, `profileUrl`, `userName` をベストエフォート取得
3. 失敗時は `warnings` を返す
4. UI で `user_name` 入力欄を強調し、手動補完しやすくする

### 追加改善案

- `user_name` 未解決時だけ入力欄をハイライトする
- `profileUrl` をワンクリックで開けるようにする
- `userId` から profile ページを再解決する再試行ボタンを用意する
- 「表示名が取得できない場合があります」と明示する

### メリット

- 今の構成を大きく変えずに進められる
- 実装コストが低い
- サーバー側だけで完結しやすい

### デメリット

- Threads / Instagram は表示名取得が不安定
- サービス側の HTML 変更に弱い
- 完全自動化は期待しづらい

### この方式が向いているケース

- まずは業務フローを止めずに使いたい
- 多少の手入力は許容できる
- ブラウザ拡張や自動化は避けたい

## 推奨

要件が「投稿 URL を貼るだけで表示名まで自動取得したい」に強く寄っているなら、Threads / Instagram は方式1に寄せたほうがよい。

逆に、まず運用を始めることが優先なら、方式2のまま UI 補助を強くするのが現実的。

---

## 現時点の結論

- X: 投稿 URL から表示名取得が通る
- Threads: 投稿 URL から `userId` は取れるが、表示名は安定しない
- Instagram: 投稿 URL から `userId` を補える場合はあるが、表示名は安定しない
- TikTok: handle ベースの解決はある程度できるが、表示名取得は未強化

# Constraints

## GitHub Pages 前提の制約

- 配布形態は GitHub Pages を前提にしている
- 静的公開しやすい構成を優先している
- サーバー常駐前提の機能は採用しにくい

## データ取得まわりの制約

- 投稿 URL から取得できる情報はプラットフォームごとにばらつきがある
- 表示名は X 以外で安定しないケースがある
- `表示数` `再生数` `いいね数` は手動入力前提
- 公開 HTML や公開メタデータに依存する部分は外部サービスの変更影響を受ける

## UI と運用の前提

- 自動取得に失敗した値は手動で直す前提
- warning 表示は、補完が必要なケースを知らせる補助として使う
- テンプレート生成は可変文章生成ではなく、固定文面の差し込み方式

## 開発運用メモ

- `npm run dev` と `npm run build` を同時並行で触ると `.next` が壊れることがある
- 開発サーバーが不安定になった場合は `.next` を削除して再起動すると復旧しやすい
